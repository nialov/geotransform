geotrans.transform module¶

Description: Module in charge of all loading and saving.

geotrans.transform.check_file(filepath: pathlib.Path) → None¶

Checks that filepath is valid and file is valid but not if its a geodata file.

geotrans.transform.determine_filetype(filepath: pathlib.Path) → str¶

Takes a single geodata filepath and determines its geodata type.

geotrans.transform.load_file(filepath: pathlib.Path) → Tuple[List[geopandas.geodataframe.GeoDataFrame], List[str]]¶

Takes a single geodata filepath string, determines its type and returns a list of GeoDataFrames and a list of layer names that consist of all file data.

geotrans.transform.load_multilayer(filepath: pathlib.Path) → Tuple[List[geopandas.geodataframe.GeoDataFrame], List[str]]¶

Use fiona to parse geopackage and filegeodatabase layers. Returns a list of GeoDataFrames and a list layer names.

geotrans.transform.load_singlelayer(filepath: pathlib.Path, filetype: str) → Tuple[List[geopandas.geodataframe.GeoDataFrame], List[str]]¶

Load a single layer from a geodata file.

geotrans.transform.save_files(geodataframes: List[geopandas.geodataframe.GeoDataFrame], layer_names: List[str], filenames: List[pathlib.Path], transform_to_type: str) → None¶

The user can save his geodata files with multiple different options.

If a directory is given, the geodata files are saved to that directory with the acquired layer names. If a single filename is given all geodata is saved to that file and this option requires checking that the format can handle multiple layers i.e. if shapefile is the save option: it cannot handle multiple layers and an error is thrown.

Filenames is a list of Paths that have been checked to be valid. If user inputted no filenames: the filename will be made from layer name before this function.

geotrans.transform.validate_loaded_geodataframes(geodataframes, layer_names)¶

Some columns (namely FID) cause issues when saving to geopackage. Currently these columns are removed to remove the issue.
